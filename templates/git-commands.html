Git Commands Cheat Sheet

üöÄ Project Workflow: Create & Push to GitHub

Follow these steps to initialize a Git repository and push your project to GitHub.

# 1. Navigate to your project folder
cd path/to/your-project

# 2. Initialize a Git repo
git init

# 3. Add files to staging
git add .

# 4. Commit changes
git commit -m "Initial commit"

# 5. Create a repo on GitHub (manual step)

# 6. Link local repo with remote
git remote add origin https://github.com/yourusername/repo-name.git

# 7. Push your code to GitHub
git push -u origin main

üîß Setup & Configuration

Set your name, email, default editor, and list global settings.

git config --global user.email "your@email.com"
git config --list
git config --global core.editor "code --wait"

üìÅ Create & Clone

üíæ Staging & Commit

Track and save changes to your project.

üîÅ Branching

Create, switch, and delete branches.

üîÑ Merging & Rebase

Integrate changes from another branch or rebase your commits.

üöÄ Push & Pull

Send your changes to GitHub or fetch new changes.

üìú Log & Diff
